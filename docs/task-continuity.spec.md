# Spec: Task Continuity — Memory, Archive & Git Operations

## Overview

Three problems solved together:

1. **Worktree longevity** — keep the environment alive through QA cycles,
   never auto-delete until user explicitly closes it
2. **Persistent memory** — a concise `memory.md` per task that survives
   worktree deletion, searchable months later when a bug surfaces
3. **Flexible storage** — committed to repo for owned projects, namespaced
   externally for shared/client repos

---

## Storage model

### Two modes based on repo ownership

**Owned repos** (`commitLacc: true` in config)

`.lacc/` lives inside the repo and is committed to git. Task artifacts
become part of the repo's history. No external storage needed.

```
<repo>/
  .lacc/
    config.yml
    tasks/
      <taskId>/
        .spec.md
        .plan.md
        .review.md
        memory.md
        workflow.yml
    templates/
      spec-context.md
    workflows/
      feature-development.yml
    .claude/
      commands/
        spec.md             ← repo-specific command overrides
```

Archive = git. `git log .lacc/tasks/` shows the full AI-assisted work
history. Push to remote and it's backed up permanently.

**Shared/external repos** (`commitLacc: false`, default)

`.lacc/` lives outside the repo, namespaced by remote URL.
No gitignore needed. Nothing added to the shared repo.

```
~/.lacc-data/repos/
  github.com/
    acme-corp/
      frontend/
        <taskId>/
          .spec.md
          .plan.md
          .review.md
          memory.md
          workflow.yml
      backend/
        <taskId>/
  gitlab.com/
    client/
      their-app/
        <taskId>/
```

Namespace derived automatically from `git remote get-url origin`.
No manual config needed.

### Namespace derivation

```typescript
function getTaskStoragePath(repoPath: string, taskId: string): string {
  const config = loadRepoConfig(repoPath);

  if (config.commitLacc) {
    // Owned repo — store inside worktree
    return path.join(worktreePath, '.lacc', 'tasks', taskId);
  }

  // External repo — derive namespace from git remote
  const remote = execSync('git remote get-url origin', { cwd: repoPath })
    .toString().trim();

  const namespace = parseRemoteToNamespace(remote);
  // github.com/acme-corp/frontend → github.com/acme-corp/frontend
  // git@github.com:acme/repo.git → github.com/acme/repo

  return path.join(
    os.homedir(), '.lacc-data', 'repos', namespace, taskId
  );
}

function parseRemoteToNamespace(remote: string): string {
  // Handle SSH: git@github.com:org/repo.git
  const ssh = remote.match(/git@([^:]+):(.+?)(?:\.git)?$/);
  if (ssh) return `${ssh[1]}/${ssh[2]}`;

  // Handle HTTPS: https://github.com/org/repo.git
  const https = remote.match(/https?:\/\/([^/]+)\/(.+?)(?:\.git)?$/);
  if (https) return `${https[1]}/${https[2]}`;

  // Fallback: use local path slug
  return path.basename(repoPath);
}
```

If no git remote exists (local-only repo), fall back to local path slug.

## Container mounting

For both modes, the agent sees `.lacc/` at a consistent path inside
the container: `/workspace/.lacc/`

**Owned repo** — `.lacc/` is literally in the worktree, so the mount
already includes it via `/workspace`. No extra mount needed.

**External repo** — mount the task storage folder alongside the worktree:

```typescript
Binds: [
  `${worktreePath}:/workspace`,
  `${taskStoragePath}:/workspace/.lacc`,   // appears inside workspace
  `${home}/.claude:/home/node/.claude`,
  `${home}/.ssh:/home/node/.ssh:ro`,
  `${repoPath}:/original-repo:ro`,
  `${home}/.lacc-data/.claude:/lacc-global:ro`,
]
```

From the agent's perspective, `.lacc/` always appears inside `/workspace`
regardless of where it actually lives on the host.

## Memory snapshot

A command/button that distils the current task state into a concise
`memory.md`. Generated by `claude -p`, not a raw dump.

### What it produces

```markdown
# feat/auth-jwt — Memory Snapshot
Generated: 2026-03-24 14:32

## What was built
JWT refresh token flow. Added /auth/refresh endpoint, token rotation
strategy, and middleware guards on protected routes.

## Key decisions
- 7-day refresh token expiry (discussed with user, original spec said 1h)
- httpOnly cookies over localStorage for security
- Refresh tokens stored in Redis with user-id index for revocation

## Files changed
- src/auth/jwt.ts (+82 / -12) — core token logic
- src/auth/middleware.ts (+34) — route guards
- src/auth/refresh.ts (+67) — new endpoint
- tests/auth.test.ts (+145) — full test coverage

## Review findings
- Error messages too generic → fixed
- Missing rate limiting on refresh endpoint → added
- Token cleanup on logout not implemented → deferred (ENG-445)

## Deferred / known issues
- ENG-445: Token cleanup on logout
- Mobile Safari cookie issue needs investigation

## Context for future work
Schema in db/migrations/004_refresh_tokens.sql.
Middleware expects Authorization header OR cookie, not both.
```

### Generation

```typescript
async function generateMemorySnapshot(task: Task): Promise<string> {
  const taskDir = getTaskStoragePath(task.repoPath, task.id);
  const diff = await getDiff(task.worktreePath, task.baseBranch);

  const context = [
    `Original prompt: ${task.prompt}`,
    readIfExists(path.join(taskDir, '.spec.md')),
    readIfExists(path.join(taskDir, '.plan.md')),
    readIfExists(path.join(taskDir, '.review.md')),
    diff.files
      .map(f => `${f.path} (+${f.insertions}/-${f.deletions})`)
      .join('\n'),
    extractUserMessages(task.id),
  ].filter(Boolean).join('\n\n---\n\n');

  const prompt = `
Summarise this completed development task for future reference.
Produce a concise memory.md covering:
- What was built (2-3 sentences)
- Key decisions made and why
- Files changed (from diff)
- Review findings and how addressed
- Deferred work or known issues
- Context a future developer needs to continue

Be concise. Reference document, not a report.

Context:
${context}
`;

  return new Promise((resolve) => {
    let output = '';
    const proc = spawn('claude', ['-p', prompt], {
      env: { ...process.env },
    });
    proc.stdout.on('data', (d: Buffer) => output += d.toString());
    proc.on('close', () => resolve(extractAssistantText(output)));
    proc.on('error', () => resolve(''));
  });
}
```

### Where it's saved

Saved to `<taskStoragePath>/memory.md` — the single source of truth.

For owned repos: `<worktree>/.lacc/tasks/<taskId>/memory.md` — committed
to git alongside the other task artifacts.

For external repos: `~/.lacc-data/repos/<namespace>/<taskId>/memory.md`
— permanent, never auto-deleted, user manages manually.

## Git operations

Surfaced as action bar buttons. Run via simple-git on the host.

### API

```
POST /tasks/:id/git/rebase      Body: { branch?: string }
POST /tasks/:id/git/pull
POST /tasks/:id/git/push
POST /tasks/:id/git/reset       Body: { hard?: boolean }
POST /tasks/:id/git/stash
POST /tasks/:id/git/stash/pop
```

### Conflict handling

```json
{
  "ok": false,
  "conflict": true,
  "conflictedFiles": ["src/auth/jwt.ts"],
  "message": "Rebase conflict in 1 file. Resolve manually or reset."
}
```

UI shows conflict banner:
```
⚠ Rebase conflict · src/auth/jwt.ts
[Open in editor]  [Abort rebase]  [Reset to base]
```

## Notes

- Memory generation costs ~$0.02-0.05 (haiku-class call)
- Multiple snapshots overwrite previous — only latest kept
- For owned repos: full task history visible in git log
- For external repos: archive never auto-deleted, user manages manually
- Git operations run on host via simple-git, not inside container
- Rebase conflicts must be resolved in IDE — LACC surfaces them only
- Branch never deleted on Close task — user pushes first if needed
- `docs/` folder convention is user-defined — LACC reads it as context
  but never writes to it
